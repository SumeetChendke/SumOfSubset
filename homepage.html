Sum of Subset

  1. Start with the given set of integers and a target sum.
  2. Begin with an empty subset and a running sum of 0
  3. Consider each element in the set one by one
  4. For the current element, first include it in the subset and add its value to the running sum
  5. If the running sum equals the target, record the subset as a solution.
  6. If the running sum exceeds the target or all elements have been considered, backtrack by removing the last element and try the next possibility.
  7. Repeat the process until all possible subsets have been explored
  8. Return all subsets whose sums equal the target

Analysis

The Sum of Subset problem using backtracking has a worst-case time complexity of O(2ⁿ), because in the worst case all possible subsets of the set need to be explored. Backtracking helps to prune branches where the partial sum exceeds the target, which can improve efficiency in practice. For small target sums, dynamic programming can provide a more efficient solution with polynomial time complexity.
